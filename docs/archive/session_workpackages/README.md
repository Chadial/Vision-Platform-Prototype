# Archived Session Work Packages

## Purpose

This folder stores completed session work-package documents after their active recovery role has ended.

Archive a work package here when:

- the branch scope has been completed
- the documented task is no longer the active recovery target
- the relevant status and roadmap documents already reflect the finished work

## Archiving Rule

Before moving a work package here:

1. make sure the active recovery links in `docs/SESSION_START.md` and `docs/STATUS.md` are removed or updated
2. ensure the final outcome is reflected in the relevant permanent docs
3. keep the archived filename stable so older references remain understandable

## What An Archived Work Package Should Contain

An archived work package should still be readable as a compact historical handoff.

Prefer keeping:

- the original `Purpose`
- the branch name
- the scoped execution plan
- the validation expectations
- the final outcome or closure note

If useful, add a short header note such as:

- `Status: completed and archived after docs/STATUS.md update`
- `Status: closed with partial completion; follow-up moved to ...`
